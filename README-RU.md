# Paseka

[English](README.md) · [Документация](https://russ-p.github.io/paseka/README-RU/)

Paseka — локальный CLI-рантайм для запуска AI-агентов внутри git-репозитория. Он помогает соло-разработчику инициализировать проект (колонию), запускать агентов (пчёл), хранить состояние рантайма и обмениваться событиями через шину сообщений.

Язык продукта — метафора улья: **colony** (git-проект) из **пчёл** / bees (агентов) работает в **hive** (рантайме), координируясь танцами в **air** (шине событий), а не через центральный мозг-королеву. Словарь: [bee glossary](docs/idea/glossary.md).

Это исследовательский эксперимент: может ли рой скромных агентов координироваться одной хореографией — без центрального мега-мозга — при настолько низких требованиях к каждой пчеле, что часть из них способна работать на local LLM на потребительском железе? Cloud-модели остаются там, где без них действительно не обойтись.

## Что умеет проект

- Выполнять задачи по коду — писать и ревьюить — запуская Cursor, Pi или Claude с фокусированным промптом под роль пчелы.
- Гонять AFK (headless) диспатчи и интерактивные HITL-сессии в одной колонии.
- Управлять и наблюдать за роем из CLI или локальной веб-консоли Queen Console.

## Как запустить

1. Склонировать репозиторий:

```
git clone https://github.com/russ-p/paseka.git
cd paseka
```

2. Скачать [релизный бинарник](https://github.com/russ-p/paseka/releases) для Linux или macOS либо собрать из исходников:

```
go mod download
go build -o paseka ./cmd/paseka
```

3. Инициализировать колонию и запустить рантайм:

```
./paseka init
./paseka run
```

4. Запустить локальную веб-консоль Queen Console:

```
./paseka console
```

Откройте http://127.0.0.1:8787 в браузере. Queen Console пока не требует аутентификацию, поэтому оставляйте её доступной только на localhost или другом доверенном интерфейсе. Используйте `--addr`, только если понимаете риск внешнего доступа.
